# L1 · 問題與目標 — 我們到底在解決什麼

> 這一層完全白話,不需要任何技術背景。讀完你會懂:這專案要幹嘛、為什麼這樣設計。

## 一、要解決的問題

想像一架無人機在天上飛,我們希望它能**自己看懂畫面裡有什麼** —
- 天上有沒有**別的無人機**(可能是入侵者 / 要追蹤的目標)
- 地上有沒有**人、車**

而且要**在無人機上當場算出來**,不是把影像傳回地面再分析。

這類應用常見於:反無人機(counter-UAV)、邊境/廠區巡邏、搜救、空中監看。

## 二、四個關鍵設計選擇,以及「為什麼」

這個專案的每個技術選擇,都是被「無人機」這個場景逼出來的。理解這四個「為什麼」,就理解了整個專案的骨架。

### 為什麼用「熱影像」而不是一般相機?

熱像儀拍的是**溫度**,不是可見光。好處:
- **晚上、逆光、煙霧、薄霧** 都能看 — 一般相機在這些情況等於瞎了。
- 無人機、人、車的引擎/體溫都比背景熱 → 在熱影像裡是**亮點**,反而好認。

代價:解析度低、沒有顏色、影像是灰階 — 這會影響後面模型怎麼設計(見 L2)。

### 為什麼要「機上即時」算,而不是傳回地面?

- **延遲**:傳回地面再算,一來一回可能好幾百毫秒;追一台飛行中的無人機,慢一拍就丟了。
- **通訊可能斷**:無人機飛遠、被干擾,連線就沒了;機上能算才不依賴通訊。
- **頻寬**:即時傳高解析影像很吃頻寬,機上只回傳「結果」(哪裡有什麼)就省很多。

這叫 **edge AI**(邊緣運算)— 運算放在資料產生的地方,不送雲端。

### 為什麼用「NPU 晶片」(KL730),不用一般 CPU 或 GPU?

無人機有兩個硬限制:**電** 和 **重量**。
- GPU 強但又耗電又重又熱 — 無人機載不動、飛不久。
- CPU 省電但算神經網路太慢 — 跟不上即時。
- **NPU**(神經網路專用晶片)是折衷:專為這類運算設計,**低功耗、輕、夠快**。KL730 跑我們的模型實測約 **2 瓦、43 fps**,這正是無人機要的甜蜜點。

### 為什麼用「YOLO / YOLOX」這個模型?

辨識「畫面裡有什麼、在哪裡」叫**物件偵測**。YOLO 系列的特點是**一次看完整張圖就給答案**(You Only Look Once),速度快 — 適合即時。YOLOX 是其中一個版本,選它的具體原因(授權乾淨、晶片官方支援)見 L2 與 PLAN。

## 三、目標明確化

| 項目 | 內容 |
|---|---|
| 輸入 | 無人機熱像儀的即時畫面 |
| 輸出 | 畫面中每個目標的「框 + 類別 + 信心」,類別為 **uav / person / vehicle / bicycle** |
| 跑在哪 | 無人機機上的 Kneron KL730 |
| 限制 | 低功耗(~瓦級)、輕(酬載估 170–290g)、即時(目標 ≥25 fps) |

## 四、難在哪(這決定了後面所有工作)

1. **熱影像和一般照片差很多** → 不能直接用網路上現成的模型,要用熱影像資料重新訓練。
2. **無人機在熱影像裡很小** → 遠距離時可能只有十幾個像素,難偵測(小目標問題)。
3. **要塞進 NPU** → 模型必須「量化」(壓成 INT8)才跑得動,壓的過程可能掉精度,要小心驗證。
4. **沒有現成資料** → 要拼湊多個公開熱影像資料集,而且它們視角、格式都不一樣。

> 這四個難點,分別對應後面技術鏈的:**資料、模型設計、量化、資料整合**。
