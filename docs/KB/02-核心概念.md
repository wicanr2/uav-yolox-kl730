# L2 · 核心概念 — 六個你必須放在腦中的詞

> 用比喻把六個核心概念講清楚。懂這六個,就能跟工程師對話。
> 每個概念附「想更深?」指向 L4 專文。

## 概念 1:物件偵測 (Object Detection)

**是什麼**:給一張圖,框出「哪裡有東西、是什麼」。
- 比「分類」多一步 — 分類只回答「這張圖是貓」,偵測回答「左上角有隻貓、右下角有隻狗」。
- 輸出 = 一堆 **框 (bounding box) + 類別 + 信心分數**。

**怎麼評分好壞**:用 **mAP**(平均精度),0~1 越高越好,同時看「框得準不準」和「該找的有沒有找到」。

## 概念 2:YOLO — 一眼看完

**是什麼**:一種物件偵測模型家族。名字 You Only Look Once = **整張圖一次算完**就給所有框,不像舊方法要掃描很多次。所以**快**,適合即時。

**比喻**:舊方法像拿放大鏡在圖上一塊塊找;YOLO 像一眼掃過整張圖直接圈重點。

**我們用 YOLOX**:YOLO 的一個版本(2021)。為什麼是它而不是更新的?
- 授權乾淨(Apache-2.0,可商用);很多新版是 AGPL,商用要付費。
- Kneron 晶片官方驗證過它能跑。
- 「越新越好」是迷思 — 最新的 YOLOv10/v11 用了新結構,在 NPU 上量化會出問題(我們實測 + 官方論壇都有案例)。

> 想更深(它內部三段式結構、怎麼訓練)→ [../yolox-explained.md](../yolox-explained.md)

## 概念 3:熱影像當資料

**是什麼**:熱像儀輸出的是溫度圖,通常是**單通道灰階**(一般照片是 RGB 三通道)。

**踩雷點**:幾乎所有現成模型都吃三通道。處理法:把灰階**複製成三份**當假 RGB。聽起來土,但這是業界主流且對 NPU 最不易出錯的做法。

**另一個關鍵**:熱像儀的原始資料常是 16-bit(溫度精度高),要先壓成 8-bit(AGC)才能餵模型 / 顯示。

## 概念 4:NPU — 為神經網路而生的晶片

**是什麼**:專門算神經網路的晶片。CPU 什麼都能算但慢;GPU 快但耗電重;NPU 只算神經網路這一種事,所以能做到**又快又省電又輕**。

**比喻**:CPU 是瑞士刀(萬用但每樣普通),NPU 是專業切肉刀(只切肉,但切得又快又省力)。無人機要省電省重,所以選專業刀。

**代價**:NPU 只認得固定的一組運算(operator),而且通常只吃 8-bit 整數(見概念 5)。模型必須「守規矩」才跑得動。

> 想更深(NPU command、operator 清單、模型怎麼塞進去)→ [../kneron-npu-deploy-notes.md](../kneron-npu-deploy-notes.md)

## 概念 5:量化 (Quantization) — 把模型壓成整數

**是什麼**:模型訓練時用 32-bit 浮點數(很精確但又大又慢)。要上 NPU,得壓成 **8-bit 整數 (INT8)** — 體積變 1/4、速度大增。

**比喻**:像把一張高解析照片存成低容量 JPEG。檔案變小、傳得快,但壓太狠會失真。量化就是「在可接受的失真內把模型變小變快」。

**怎麼壓**:餵一批代表性圖片(**校正集**)給工具,讓它觀察每層數值的範圍,決定怎麼把浮點對應到 256 個整數格。

**這是整個專案最大的坑**:壓的過程可能掉精度,甚至崩潰(輸出全變常數)。我們就**真的踩到**了 — 而且查了 7 輪才找到根因(範圍估計法用錯)。這段是最好的學習材料 → [../session-walkthrough.md](../session-walkthrough.md) §6。

## 概念 6:訓練 vs 部署 的鴻溝

**是什麼**:模型有兩個世界:
- **訓練世界**(PC/GPU,PyTorch):用資料把模型教會,產出「權重」。
- **部署世界**(晶片,C/SDK):把教好的模型搬上 NPU 即時跑。

**橋樑**:中間靠 **ONNX**(通用模型格式)交換,再經晶片廠的工具鏈量化、編譯成晶片專屬可執行檔(Kneron 叫 **NEF**)。

**比喻**:訓練是「寫好程式原始碼」,部署前要「編譯成這台機器的執行檔」。ONNX 是中間的標準原始碼格式,NEF 是 KL730 專屬的執行檔。
